# Academic-Data-Agent 项目概念审计

这份文档只回答一个问题：

> 当前项目到底已经实现了什么，哪些还只是未来方向？

## 1. 判断标准

只有满足下面两个条件，才能算“当前版本正式实现”：

1. 已经进入当前正式主链
2. 会对最终结果、trace 或审稿产生正式影响

因此：

- 目录存在，不等于能力已实现
- 试验过，不等于当前默认入口在用
- 保留兼容模块，不等于它仍是正式主线

## 2. 当前可以稳定宣称的能力

### Agent 工作流

可以说：

- 项目基于 `run_analysis(...)` 驱动一条受控分析工作流
- 有显式状态和可回放的运行轨迹

不要说：

- 通用自治 Agent 平台

### 表格分析主线

可以说：

- 当前正式主线支持结构化表格数据输入
- 支持从上传到报告、审稿、历史问答的一条闭环

不要说：

- 通用多模态输入平台

### RAG

可以说：

- 已有本地知识库
- 已实现 `query rewrite + hybrid retrieval + structured chunking + rerank`
- 已支持 evidence register

不要说：

- 最前沿研究级 RAG 平台

### 证据归因

可以说：

- 已有 evidence register
- 报告中的知识性解释要求带短引用
- reviewer 会检查缺引、伪引和证据错配

不要说：

- 完整学术引文系统

### Project Memory

可以说：

- 已有项目级记忆
- accepted run 会被精炼写回
- 分析前和审稿前都可回忆

不要说：

- 完整长期会话记忆网络

### 历史问答

可以说：

- 已有独立历史问答页面
- 已支持围绕历史运行继续追问
- 已支持跨运行对比总结

不要说：

- 通用聊天问答 Agent

## 3. 当前还没正式进入主链的方向

- 通用多工具平台化
- 通用聊天 Agent
- 大而全的长期记忆系统
- OCR 型 PDF 理解主线
- 重型基础设施层扩展

## 4. 当前最稳的项目描述

当前最稳妥的项目描述是：

> 一个基于 `hello-agents` 二次开发的学术数据分析 Agent 系统，正式主线聚焦结构化表格分析，并结合工程化 RAG、证据归因、Reviewer 治理、Project Memory、历史记录回放和历史问答。

## 5. 一句话结论

这个项目已经足够被描述成“有工程深度的分析型 Agent 系统”，但不应该再包装成“通用智能体平台”。
