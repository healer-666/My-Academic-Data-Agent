# Academic-Data-Agent 项目改进路线图

这份路线图只关注当前版本之后最值得继续做的方向，不回顾旧版本。

## 1. 当前状态判断

当前项目已经具备比较完整的工程闭环：

- PDF ingestion
- 受控分析 workflow
- 工程化 RAG
- 证据归因
- reviewer 治理
- project memory
- Web 工作台与历史回放

因此下一步重点不应该是继续堆概念，而应该是：

1. 提升可评估性
2. 提升可信度
3. 提升展示完成度

## 2. 第一优先级：RAG 评测体系

最值得补的一步不是继续加新模块，而是一套稳定评测。

建议补齐：

- 固定测试数据集
- 固定知识文档集
- 固定 query 集
- 无 RAG / dense-only / hybrid / structured / memory-on 的对比

为什么优先：

- 现在系统已经足够复杂，后续改 RAG 不能只凭感觉。
- 这对答辩、面试和简历都很有说服力。

## 3. 第二优先级：更细的引用与证据校验

当前已经有 evidence register 和 reviewer 校验，但还可以继续增强：

- 更细粒度的句级引用覆盖分析
- 报告段落到 citation label 的更强对齐
- reviewer 输出更结构化的 citation findings
- Web 端更清晰的 evidence summary

这条线的价值在于：

- 它能明显拉开和普通 RAG demo 的差距。

## 4. 第三优先级：Memory v2

当前 project memory v1 已经可用，下一步建议做得更稳，而不是更杂。

建议方向：

- 明确区分 `user_preference / review_constraint / analysis_pattern / rag_usage_note`
- 支持查看某个 scope 的 memory 摘要
- 增加 rejected run lessons，但和 accepted memory 分开
- 提高 memory 提取与摘要质量

暂时不建议：

- 把完整历史报告全文塞进 memory
- 做模糊的大杂烩长期记忆池

## 5. 第四优先级：PDF / 表格结构增强

这个方向仍然值得继续做，但已经不是最急的。

可选增强：

- 更好的 section-aware chunking
- caption / table note 的更细抽取
- 表头语义和数值列语义增强
- 面向科研 PDF 的 chunk 质量评估

## 6. 第五优先级：答辩与展示材料

如果项目要用于保研、简历或答辩，这部分同样重要。

建议补齐：

- 一张完整架构图
- 一张运行时序图
- 一张 RAG / evidence / memory 分层图
- 一页“当前边界与未来工作”

## 7. 当前不建议优先做的事

- MCP 总线
- 通用聊天问答模式
- 重型向量数据库基础设施
- 平台化多租户架构

## 8. 一句话结论

下一步最值得做的，不是再加一个花哨功能，而是把“评测、可信度和展示材料”补齐。
