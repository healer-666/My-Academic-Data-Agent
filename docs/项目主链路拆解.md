# Academic-Data-Agent 项目主链路拆解

这份文档只回答一个问题：

> 当前版本的主链到底怎么跑。

## 1. 主链一句话概括

```text
输入文件 -> ingestion / context -> memory recall -> RAG retrieval -> analyst loop -> report / trace -> review -> memory writeback
```

## 2. 起点：`run_analysis(...)`

真正的主入口是 [`run_analysis(...)`](/C:/Users/pc/OneDrive/Desktop/agent/src/data_analysis_agent/agent_runner.py)。

它负责：

- 加载运行时配置
- 建立 `RunContext`
- 处理 PDF ingestion
- 构建 `data_context`
- 触发 project memory recall
- 触发 RAG retrieval
- 运行 analyst loop
- 保存 report / trace
- 触发 review
- accepted run 写回 project memory

## 3. 第一阶段：输入标准化

### 表格输入

如果输入是 `csv / xls / xlsx`：

- 直接进入 `data_context`
- 不需要 `document_ingestion`

### PDF 输入

如果输入是 PDF：

- 先进入 [`document_ingestion.py`](/C:/Users/pc/OneDrive/Desktop/agent/src/data_analysis_agent/document_ingestion.py)
- 提取背景文本
- 识别 candidate tables
- 选择主表
- 生成 `parsed_document.json`

## 4. 第二阶段：上下文构建

当前上下文不是一份大 prompt，而是几层拼装：

### `data_context`

来自 [`data_context.py`](/C:/Users/pc/OneDrive/Desktop/agent/src/data_analysis_agent/data_context.py)：

- schema
- shape
- head sample
- warning
- PDF 背景摘要
- 候选表摘要

### `memory_context`

来自 [`memory/service.py`](/C:/Users/pc/OneDrive/Desktop/agent/src/data_analysis_agent/memory/service.py)：

- 项目级偏好
- 历史稳定约束
- 已接受运行提炼出的经验

### `knowledge_context`

来自 [`knowledge_context.py`](/C:/Users/pc/OneDrive/Desktop/agent/src/data_analysis_agent/knowledge_context.py)：

- User Intent
- Project Memory
- Reference Context
- Retrieved Knowledge
- Retrieved Evidence Register

## 5. 第三阶段：RAG

当前 RAG 已经是正式主链的一部分。

检索链大致是：

```text
query builder -> dense retrieval -> keyword retrieval -> merge -> rerank -> final chunks -> evidence register
```

并且已经支持：

- structured chunking
- PDF table candidates 增强
- evidence attribution

## 6. 第四阶段：analyst loop

analyst loop 的职责是：

- 基于结构化上下文做分析决策
- 调用工具
- 回收 observation
- 生成最终报告

当前不是“无限自由探索”，而是受控多步分析流程。

## 7. 第五阶段：report / trace

分析完成后会落地：

- `final_report.md`
- `agent_trace.json`
- figures
- review logs
- 可下载的工件包

trace 里会保留：

- workflow state
- event stream
- RAG payload
- evidence coverage
- memory retrieval / writeback
- review history

## 8. 第六阶段：review

review 分两层：

- 文本 reviewer
- 可选 visual reviewer

review 不只检查语言质量，也检查：

- 结论是否稳定
- 图表是否可读
- 引用是否缺失
- 证据是否匹配

## 9. 第七阶段：memory writeback

只有 accepted run 会进入长期 memory：

- 抽取项目级经验
- 精炼成 record
- 按 scope 写入 project memory

这一步保证 memory 尽量少噪音。

## 10. 一句话结论

当前主链最重要的特点不是“模型回答了什么”，而是：

> 输入、检索、分析、证据、审稿、记忆已经被串成了一条可追踪、可治理的完整流程。
